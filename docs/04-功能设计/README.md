## 🎯 使用说明

`04-功能设计/` 是当前仓库的模块级详细设计目录。

阅读顺序建议如下：

1. 先读 `../00-总设计/总体项目设计.md`
2. 再读本文件
3. 再读 `M901-功能模块关系与开发顺序设计.md`
4. 再按"平台 -> 场景"的顺序进入单模块文档
5. 涉及页面实现或 UI 评审时，再读 `../13-界面设计/README.md`

## 📝 编写约束

所有模块文档必须遵守：

1. 写清这个模块属于平台、CVE 还是安全公告。
2. 写清主流程、异常流程、接口入口、数据对象和测试要点。
3. 写清依赖模块和非目标范围。
4. 不复活旧 `pipeline/product/diagnostics` 语义。
5. 任何场景结果都必须能回到证据和 Artifact。
6. 涉及前端页面的模块，必须显式回链到对应 `Pxxx` 页面文档。

## 🚀 实现阶段顺序

Phase 1 起实现顺序固定为：

1. 平台最小底座
2. CVE 补丁获取最小闭环（fast-first）
3. CVE 证据详情与主链鲁棒性增强（family / direct candidate / fallback 评估）
4. 监控、投递、完整观测
5. 安全公告 URL 模式
6. 安全公告手动提取
