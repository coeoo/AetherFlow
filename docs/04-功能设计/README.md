## 🎯 使用说明

`04-功能设计/` 是当前仓库的模块级详细设计目录。

阅读顺序建议如下：

1. 先读 `../00-总设计/总体项目设计.md`
2. 再读本文件
3. 再读 `M901-功能模块关系与开发顺序设计.md`
4. 再按“平台底座 -> CVE Patch Agent -> 安全公告”顺序进入单模块文档
5. 涉及页面实现或 UI 评审时，再读 `../13-界面设计/README.md`

## 📝 编写约束

所有模块文档必须遵守：

1. 写清这个模块属于平台、CVE 还是安全公告。
2. 写清主流程、异常流程、接口入口、数据对象和测试要点。
3. 写清依赖模块和非目标范围。
4. 不复活旧 `pipeline/product/diagnostics` 语义。
5. 任何场景结果都必须能回到证据、Artifact 与搜索决策。
6. 涉及前端页面的模块，必须显式回链到对应 `Pxxx` 页面文档。

## 📌 当前阶段重点

当前文档体系已经确认：

- CVE 场景的正式方向是 **LangGraph 编排的受控多跳 Patch Agent**
- 功能文档必须围绕搜索图、预算、页面角色、Agent 决策和 patch 收敛展开
- 不再把 `fast-first` 规则链作为长期主线进行传播

---

## 🚀 实现阶段顺序

推荐的实现顺序如下：

1. 平台最小底座与任务编排能力
2. CVE Patch Agent 图运行时与工具层
3. CVE 工作台与详情页搜索图展示
4. 搜索图持久化、预算治理与人工复核
5. 公告场景接入统一图运行时与 Artifact 基座
6. 投递、监控、完整观测与治理
