# Patch 检索单页结果中心设计

## 目标

把当前前端的 “`/cve` 工作台 + `/cve/runs/:runId` 详情页” 模型，重构为以 **`/patch` 单页结果中心** 为主的 Patch 检索体验。

本次改造只动前端，不回滚旧后端模型，也不恢复旧版 graph run 契约。当前输入模式仍然只支持 `CVE` 编号，但页面、导航和文案不再把这条业务线称为 “CVE 场景”。

## 设计原则

1. 主业务名称统一为 **Patch 检索**，`CVE` 只作为输入类型和结果字段存在。
2. `/patch` 必须是结果优先单页，而不是先操作再跳详情的工作台。
3. 有历史结果时先展示历史，只有用户主动点击 “重新检索” 时才创建新 run。
4. 普通用户信息与工程信息分层展示：
   - 结论、主补丁、证据链属于用户阅读层
   - 运行元信息、原始页面轨迹、补丁原始数据属于开发者详情层
5. `/patch/runs/:runId` 保留为深链调试页，不再是默认阅读路径。

## 路由与入口

- 主入口：`/patch`
- 深链页：`/patch/runs/:runId`
- 不保留 `/cve`

## 页面结构

`/patch` 页面按如下顺序组织：

1. 顶部检索 Hero
2. 当前结果 Hero
3. 主补丁摘要
4. 修复族与关键证据链
5. 开发者详情（可展开）
6. 最近运行

整体结构延续旧版“单页结果中心”的阅读节奏，但视觉上使用现仓样式体系重绘，而不是直接照搬旧 CSS。

## 数据策略

### 历史优先

当用户在 `/patch` 中输入合法 CVE 编号并触发查询时：

1. 先从运行列表中找同 CVE 的最近一条 run
2. 命中则直接拉取详情并展示
3. 未命中则展示“当前没有历史结果”的空态
4. 不自动创建新 run

### 主动重新检索

只有用户点击下列 CTA 时才创建新 run：

- `开始检索`
- `重新检索`

创建成功后：

1. 以新 run 接管当前结果区
2. 在当前页轮询 detail
3. run 进入终态后刷新最近运行列表

### URL 表达

建议使用轻量 query 参数表达当前查询对象：

- `/patch?q=CVE-2022-2509`

该参数只负责表达当前查询对象，不承载旧版 `/cve?q=...` 的整套产品语义。

## 组件策略

优先复用现有新版 detail payload 对应的组件：

- `frontend/src/features/cve/components/CVEVerdictHero.tsx`
- `frontend/src/features/cve/components/CVEPatchList.tsx`
- `frontend/src/features/cve/components/CVETraceTimeline.tsx`
- `frontend/src/features/cve/components/CVEFixFamilySummary.tsx`
- `frontend/src/features/cve/components/CVEDiffViewer.tsx`

新增一个单页聚合容器用于：

- 管理查询状态
- 管理历史命中 / 新 run 接管
- 统一编排结果区块
- 管理开发者详情展开状态
- 管理选中 patch 与 diff 加载

## 开发者详情

开发者详情回到 `/patch` 同页底部，默认收起，内部固定包含三个区块：

1. 运行元信息
2. 原始页面轨迹
3. 补丁原始数据

这样既能保持主页面清爽，又能恢复旧版“工程信息随时可下钻”的优势。

## 命名与文案

前端统一替换为：

- 主导航：`Patch 检索`
- 页面标题：`Patch 检索`
- 主按钮：`开始检索`
- 历史结果按钮：`重新检索`
- 深链按钮：`查看运行详情`
- 工程区标题：`开发者详情`

前端不再出现：

- `CVE 场景`
- `CVE 工作台`
- `CVE 补丁检索`

必须保留 `CVE` 的位置仅限于：

- 输入框标签
- 示例占位
- 结果数据字段
- API / 类型字段名

## 状态设计

页面必须明确支持：

1. 首页空态
2. 无历史空态
3. 历史结果态
4. 主动重检加载态
5. 业务失败态
6. Diff 局部加载态
7. 局部接口失败态

关键约束：

- 历史命中时不自动重新检索
- 重检时旧结果不闪空
- 业务失败不是页面失败
- Diff 加载只影响局部区块

## 风险与边界

1. 本次改造会显著调整前端状态机，不再只是基于 `activeRunId` 驱动页面。
2. `/patch` 与 `/patch/runs/:runId` 的职责边界会重写，但后端接口保持不变。
3. 测试需要从“工作台 + 详情页”模型迁移到“结果优先单页 + 深链详情页”模型。
4. 当前输入模式仍然只支持 `CVE`，不提前扩展其他 Patch 检索入口。
