# CVE 检索工作台使用说明

> **如何使用 CVE 补丁检索**

---

## 1. 功能简介

该工作台用于输入一个 `CVE ID`，让系统主动检索补丁、展示运行过程，并提供可复查的补丁证据。

---

## 2. 使用步骤

1. 打开页面 `/cve`
2. 在输入框中输入 CVE，例如 `CVE-2024-3094`
3. 点击“开始检索”
4. 等待运行状态、最近进展和结果摘要更新
5. 点击“查看详情”进入证据页继续复核
6. 也可以在页面底部的“最近运行”区域回看刚刚跑过的记录

---

## 3. 页面说明

- **输入框**：仅接受合法 CVE 编号
- **运行状态卡片**：展示当前阶段与最近进展
- **结果摘要**：展示是否找到补丁、主证据与下一步操作
- **最近运行**：展示最近几次 run 的主证据或失败原因，支持直接跳详情
- **详情页入口**：进入完整 trace、patch、diff 查看页
- **失败态进度**：如果中途失败，会保留真实失败阶段，而不是显示“全部完成”

---

## 4. 常见问题

### 为什么一直显示运行中？
- 说明后台还在探索引用页面或下载 patch。
- 若长时间无进展，请查看详情页中的停止原因或联系维护者。

### 为什么没有找到补丁？
- 并非所有 CVE 都能直接命中可下载 patch。
- 详情页会展示系统探索过的页面和停止原因。

### 为什么 patch 不能查看 diff？
- 可能该候选未下载成功，或原始内容文件已不存在。
- 当前详情页内部会优先按稳定的 `patch_id` 读取 diff 内容。

### 为什么详情页里会看到“共 2 条记录”？
- 说明同一个 patch URL 在运行过程中被重复发现或重复落表。
- 页面只展示一条代表记录，但会保留重复数量，方便排障和后续补偿。

---

## 5. 结果解释

- **已命中补丁**：系统找到了可用 patch/diff，并保留了内容或下载结果。
- **Patch 可查看**：按钮可点，说明 Artifact 与磁盘文件都仍然存在。
- **共 N 条记录**：同一 `candidate_url` 有多条落表记录，页面已合并展示。
- **未命中**：系统完成探索，但没有找到满足规则的 patch。
