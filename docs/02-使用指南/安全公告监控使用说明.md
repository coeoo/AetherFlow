# 安全公告监控使用说明

> **如何配置和使用安全公告监控**

---

## 1. 功能简介

安全公告监控当前用于查看监控源的抓取批次、批次详情和关联公告 run。

当前文档按仓库已实现事实保守描述：

- 已具备监控源管理入口与监控批次查看页
- 已具备批次到公告 run 的联动查询
- 自动调度频率、不同来源适配深度和完整投递策略仍在持续完善中

---

## 2. 配置步骤

1. 打开 `/announcements/sources`
2. 点击“新增监控源”
3. 选择来源类型
4. 填写抓取配置、调度周期和通知策略
5. 保存
6. 点击“立即试跑”验证
7. 自动跳转到 `/announcements?tab=monitoring` 查看本次批次

---

## 3. 运行说明

- **启用状态**：启用后，由 scheduler 按当前已配置能力参与监控执行；完整自动调度语义仍在持续完善
- **立即试跑**：不会等待调度周期，立即创建一次抓取任务
- **批次记录**：每次运行都会保留抓取批次记录和新增结果
- **两层结果**：先看监控批次，再从批次进入单文档情报包详情

---

## 4. 页面怎么查看

### 监控源管理页
- 路径：`/announcements/sources`
- 负责配置来源、启停和立即试跑

### 监控批次页
- 路径：`/announcements?tab=monitoring`
- 负责查看某次抓取批次抓到了多少内容、触发了哪些提取 run

### 单文档结果页
- 路径：`/announcements/runs/{run_id}`
- 负责查看某一篇公告的结构化情报包

---

## 5. 常见问题

### 试跑失败怎么办？
- 先检查配置项是否完整
- 再查看批次详情中的错误信息

### 为什么没有新增结果？
- 可能确实没有新内容
- 也可能被去重逻辑判定为已处理过

### 为什么只抓到了列表，没有结果包？
- 说明抓取成功，但正文归一化或结构化提取环节失败
- 需要从该批次关联的单文档 run 继续查看失败详情
