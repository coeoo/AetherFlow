# 系统任务与状态使用说明

> **如何查看平台任务与系统状态**

---

## 1. 功能简介

系统工具页用于查看平台执行状态，而不是替代场景结果页。

包含两部分：

- `/system/tasks`：任务中心
- `/system/health`：系统状态

---

## 2. 任务中心怎么用

1. 打开 `/system/tasks`
2. 按场景、状态或触发方式筛选
3. 点击某条任务查看详情和 attempt 记录
4. 如果任务失败，可点击“重试”
5. 如果需要看业务结果，点击关联的场景 run 进入对应详情页

注意：

- 平台重试只会重试同一个 run。
- 如果你要重新发起一次新的业务执行，应回到场景页重新提交。

---

## 3. 系统状态怎么用

1. 打开 `/system/health`
2. 查看 API、数据库、Worker、Scheduler 的状态
3. 如果某个组件异常，再决定是否进入任务中心排查

---

## 4. 常见问题

### 为什么系统页是降级，不是 down？
- 表示系统还能部分工作，但关键依赖已有异常。

### 为什么任务重试后没有新的 run？
- 平台级重试只增加新的 attempt，不新建 run。
