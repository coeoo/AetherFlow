# 投递管理使用说明

> **如何管理通知目标和查看投递记录**

---

## 1. 功能简介

投递中心用于管理三类通知目标：

- 邮件
- 微信/企业微信
- Webhook

并查看每次发送的记录和失败原因。

---

## 2. 使用步骤

### 新增投递目标
1. 打开 `/deliveries`
2. 点击“新增目标”
3. 选择渠道类型
4. 填写目标信息
5. 保存目标

### 编辑投递目标
1. 打开 `/deliveries`
2. 在目标列表点击“编辑目标”
3. 修改名称、渠道类型、启用状态或 `config_json`
4. 点击“保存修改”

### 查看投递记录
1. 打开投递记录标签页
2. 按场景、状态、渠道筛选
3. 查看记录详情
4. 失败时可重试

### 记录筛选持久化
1. 在 `/deliveries?tab=records` 调整筛选条件
2. 点击“应用筛选”
3. 页面会把 `scene_name`、`status`、`channel_type` 写入 URL
4. 刷新页面或分享链接后仍能保留当前筛选组合

### 从公告结果页手动发送
1. 打开 `/announcements/runs/{run_id}`
2. 滚动到投递区块
3. 查看系统是否建议发送
4. 勾选要准备的目标
5. 点击“生成投递记录”

---

## 3. 注意事项

- 禁用目标不会参与新投递
- 测试发送和正式发送会分开记录
- 敏感配置不会在页面明文展示
- 自动发送与手动发送会共用同一记录中心，但来源动作不同
- 当前公告详情页生成的是平台内 `delivery_records`，尚未触发真实渠道发送
