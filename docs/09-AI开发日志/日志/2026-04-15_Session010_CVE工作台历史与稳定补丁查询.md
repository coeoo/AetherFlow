## 做了什么

- 为 CVE 工作台新增最近运行列表，支持直接回看最近几次 run 的状态、主证据和详情入口
- 新增 `GET /api/v1/cve/runs`，按创建时间倒序返回最近若干次运行摘要
- 详情接口 `patches` payload 新增 `patch_id`
- `GET /api/v1/cve/runs/{run_id}/patch-content` 优先切换为按 `patch_id` 读取 diff，同时保留 `candidate_url` 兼容
- 前端详情页从 `candidate_url` 切换到 `patch_id` 作为 patch 选中键
- 为工作台和详情页补了 phase / stop reason 的可读文案映射

## 关键决策

- 工作台历史列表只做最小回看，不在本轮引入筛选、分页和复用运行
- 详情页内部交互统一优先使用稳定的 `patch_id`，不再依赖 `candidate_url`
- `patch-content` 接口保留 `candidate_url` 兼容，避免历史调用立即失效
- stop reason 和 phase 先通过前端映射做可读化，不额外引入新的后端文案字段
