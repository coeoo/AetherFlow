# 2026-04-17 Session015 平台首页公告监控投递执行文档同步

## 本次目标

基于 `main` 已合入的三条能力线，统一回写仓库文档，解决“代码已经落地但文档仍写成未开始/设计中”的问题。

本次回写覆盖三类能力：

- 平台首页摘要、系统状态页、任务中心
- 安全公告监控批次与详情查询
- 投递中心目标管理、测试发送、立即发送、计划发送、失败重试与配置脱敏

## 本次判断

当前仓库已经明显进入“代码快于文档”的阶段。

需要修正的核心偏差有三类：

1. 多份总口径文档仍把平台首页、系统页、任务中心写成“未真实数据化”。
2. 安全公告监控文档仍把批次查询能力写成未开始，或反过来把完整自动调度写得过满。
3. 投递中心文档仍停留在“只有目标 CRUD / 基础查询”的早期状态，没有同步立即发送、计划发送、失败重试和配置脱敏。

## 回写原则

- 只按当前代码已证实的最小事实回写，不把目标能力当成既有事实。
- 对安全公告监控保守描述为“批次可查、详情可看、关联 run 可追”。
- 对 scheduler 保守描述为“heartbeat、监控抓取和到期投递等最小职责已落地”，不写成完整调度平台已成熟。
- 对投递中心明确写出执行动作已落地，但不宣称所有渠道都已完成生产级联调。
