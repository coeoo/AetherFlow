## 关键决策

- GitHub 下载失败的排查顺序应优先看匿名 API 限流和超时重试，而不是先假设浏览器头缺失
- 下载稳定化应放在下载器内部完成，而不是交给图节点层做外部重试
- 终态候选必须在图节点层过滤，否则会造成重复下载和证据污染
- GitHub API fallback 必须是可选能力，且明确依赖 `GITHUB_TOKEN`
