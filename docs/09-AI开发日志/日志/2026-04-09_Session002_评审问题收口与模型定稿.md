## 关键决策

- 平台任务层与场景 run 的稳定绑定通过场景表 `job_id UNIQUE` 反向建立，不再依赖 `task_jobs.result_ref_*` 弱引用。
- 平台 `/retry` 不会创建新 run，只会在同一 job 上创建新 attempt。
- 公告监控批次存在于 `source_fetch_records`，不会再让 `announcement_run` 兼任批次容器。
- 公告场景 canonical 文本只通过 Artifact 读取，业务表只保留引用和摘要字段。
- 监控去重分两层：源内幂等阻断重复执行，跨来源重复只提示不阻断。

- 处理：改为 run 表持有 `job_id UNIQUE`，冻结 retry 语义。
