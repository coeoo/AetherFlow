## 关键决策

- 详情页优先使用 `GET /api/v1/cve/runs/{run_id}` 一个接口承载完整 payload，不新增独立 `/patches` 接口
- 对历史重复 patch URL 不再承诺“最新一条”，而是统一选择“最可消费的代表条目”
- 同 URL 重复记录不重复出卡，改为通过 `duplicate_count` 结构化暴露
- 失败 run 的进度必须保留真实失败阶段，不能伪装成“全部完成”

## 问题与处理

- 失败 run 进度被错误标成 `6/6 已完成`
  - 处理：保留真实 `phase`，并在详情服务中按失败点计算 `completed_steps`
- 同一 patch URL 从多个页面被发现时，详情接口有 500 风险
  - 处理：运行链路先去重，详情服务对历史重复记录做代表条目合并
- `content_available` 与磁盘文件真实存在性不一致
  - 处理：把磁盘文件存在性纳入后端判断，再决定前端按钮是否可点
