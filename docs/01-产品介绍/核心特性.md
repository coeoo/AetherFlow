## 🎯 产品能力总览

v1 的目标不是做成一个泛化后台，而是围绕两个真实场景沉淀可复用的情报处理能力：

- **CVE Patch Agent**
- **安全公告提取**

平台层负责承载、调度、抓取、留痕与投递；
场景层负责完成具体智能体主链与结果解释。

---

## ✨ 平台共用能力

- **统一任务底座**：手动触发、定时触发、失败重试、后台执行。
- **统一文档采集基座**：抓取 URL、归一化正文、保存快照、沉淀 Artifact。
- **统一证据体系**：结果、证据片段、原文内容与附件均可回放。
- **统一投递能力**：支持目标管理、配置脱敏、测试发送、立即发送、计划发送。
- **统一平台入口**：首页、任务中心、系统状态与投递中心复用平台视图。

---

## 🧠 Patch Agent 能力

- **多源 seed 聚合**：同时消费 CVE 官方记录、OSV、GitHub Advisory、NVD
  等来源，形成首轮搜索上下文。
- **浏览器驱动型多跳搜索**：通过 Playwright 浏览器获取页面，而不是沿用旧的文本抓取主链。
- **页面角色识别**：区分 advisory、tracker、mailing list、bugtracker、
  commit、pull request、merge request、download、repository 等页面角色。
- **链路追踪与跨域推进**：围绕 `NavigationChain`、`page_role_history`、
  `cross_domain_hops` 组织跨域链路，而不是只保留平面 URL 列表。
- **LLM 主决策 + 有限保底机制**：由 LLM 决定扩展哪些链接、何时尝试下载候选，
  同时通过 validator、rule fallback、deterministic URL fallback 保证在受限边界内收口。
- **预算治理**：限制总页面数、深度、frontier、跨域扩展与下载尝试，
  保证搜索受控。
- **候选收敛**：把多个来源发现的 patch 入口聚合为可信候选族，并输出主结论。
- **补丁下载与校验**：下载 patch/diff/debdiff 或 commit patch，
  校验内容格式并保留原始内容。
- **搜索图回放**：详情页展示搜索图、frontier、预算、页面角色、链路追踪、
  决策历史、patch 收敛和 diff。
- **稳定验收与回归门禁**：通过 acceptance baseline、compare 和 regression gate
  对真实链路样本与鲁棒性边界做本地稳定验证。

---

## 📢 安全公告场景能力

- **手动提取**：支持 URL 和正文粘贴两种输入模式。
- **监控提取**：支持监控源管理、批次运行、批次详情与关联结果追踪。
- **结构化情报包**：输出受影响对象、IOC、风险判断、修复建议与证据片段。
- **结果投递**：将情报包接入邮件、企业微信或 Webhook 工作流。
- **证据留存**：保留公告原文、抓取快照、提取摘要和投递记录。

| 能力 | 平台 | Patch Agent | 安全公告 |
|------|------|-------------|----------|
| 首页与工作台入口 | ● | 复用 | 复用 |
| 后台任务执行 | ● | 复用 | 复用 |
| 定时调度 | ● | 可接入 | ● |
| 文档抓取与 Artifact | ● | 复用 | 复用 |
| 多源页面探索 | 基座支持 | ● | ● |
| 图运行时与决策审计 | 基座支持 | ● | ○ |
| 结构化结果表达 | 基座支持 | Patch 结果 | 公告情报包 |
| 结果详情页 | 基座支持 | 搜索图详情 | 情报包详情 |
| 投递链路 | ● | 可接入 | ● |

- `●` = 正式能力
- `○` = 可按场景逐步接入

## 🚫 不作为主线的方向

- 通用开放式网页代理
- 无预算、无护栏的自由浏览智能体
- 以工程调试字段为核心体验的后台页面
- 旧 `pipeline / product / diagnostics` 命名体系
- 与场景主链脱节的抽象能力堆叠
