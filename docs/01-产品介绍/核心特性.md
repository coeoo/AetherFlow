# 核心特性

> **AetherFlow v1 目标能力矩阵**

> 读前建议：先阅读 [`总体设计`](../00-总设计/总体项目设计.md)。本文同时描述 v1 的 **目标能力** 与 **当前已实现状态**。除特别注明外，不应把目标能力直接等同于当前代码事实。

---

## 🎯 版本目标

v1 的目标是建成一个有真实场景路径的情报处理平台，而非抽象框架：

- **CVE 补丁检索** — 输入 CVE 编号，主动检索并下载补丁
- **安全公告提取** — 从安全公告中提取结构化情报

承载平台要把这两条路径都承载起来，同时沉淀真正可复用的共用底座。

---

## ✨ 核心特性总览

### 平台共用能力
- **统一任务底座**：手动触发、定时触发、失败重试、后台执行。
- **统一文档采集基座**：URL 抓取、正文归一化、原文快照、Artifact 持久化。
- **统一投递能力**：目标管理、配置脱敏、测试发送、立即发送、计划发送、失败重试。
- **统一工作台入口**：平台首页统一展示场景入口、最近任务、最近投递和系统状态。
- **统一证据保留**：分析结果、证据片段、原文和附件都能复查。

### CVE 场景能力
- **多源聚合**：同时查询多个权威来源。
- **页面探索**：沿可信链接继续查找补丁候选。
- **补丁下载**：下载 patch/diff 并保留原始内容。
- **结果解释**：展示 stop reason、trace、fix family 和 patch 证据。
- **差异查看**：直接查看补丁 diff 内容。

### 安全公告场景能力
- **手动提取**：支持 URL 与正文粘贴两种入口。
- **监控提取**：支持监控源管理、监控批次查询、批次详情查看与关联公告 run 追踪。
- **结构化情报包**：输出受影响对象、IOC、风险判断、修复建议和证据片段。
- **多源适配**：当前保守描述为已具备首批监控源类型与批次结果表达，不把所有来源闭环程度写成完全一致。
- **结果投递**：平台侧已具备投递记录执行动作，场景侧生产化策略仍按目标继续收口。

## 🧩 当前已实现状态

截至 `2026-04-17`，当前仓库已确认落地的最小能力如下：

- **平台首页**：已真实消费首页摘要接口，展示场景入口、最近任务、最近投递和系统状态摘要。
- **系统状态页**：已真实消费健康摘要接口，展示 API、Database、Worker、Scheduler 与 notes。
- **任务中心**：已具备任务列表查询与 attempt 轨迹展示，不再只是路由壳。
- **安全公告工作台**：已支持手动提取 run、监控批次列表、批次详情和关联 run 跳转。
- **投递中心**：已支持目标新增/编辑、配置脱敏回显、测试发送、立即发送、计划发送、失败重试。
- **CVE 工作台**：仍是当前最完整的业务主链，具备真实运行、详情追踪与补丁证据表达。

当前仍需保守处理的边界：

- 安全公告监控已经有“批次可查”的切片，但不应写成完整自动调度平台已经成熟。
- 投递执行动作已落地，不等于所有渠道都已完成生产级适配与实发联调。
- 本文中的长期目标能力矩阵，仍然大于当前代码实现面。

---

## 🚫 当前不包含的能力

- 多租户与复杂权限体系
- 旧 `pipeline / product / diagnostics` 语义兼容
- CVE legacy lookup API
- 开放式网页代理或通用多 agent 平台
- 完整的 SaaS 运维面板
