## 🎯 这是什么

AetherFlow 是一个安全情报处理引擎，以原始安全信号为输入、结构化情报为输出。
承载平台以“平台首页 + 场景工作台”的形态承载首批场景：

- `CVE 补丁检索`：给定一个 CVE，主动检索补丁与证据。
- `安全公告提取`：对安全公告做结构化提取，并支持持续监控。

## 🧭 产品入口

### 平台首页
- 查看两个场景入口与当前阶段说明
- 查看最近任务与最近投递记录
- 查看系统状态摘要并跳转到任务中心 / 系统状态页

### CVE 工作台
- 输入 CVE 编号
- 发起实时 CVE run
- 查看运行状态、证据链、补丁和 diff

### 安全公告工作台
- 手动输入公告 URL 或正文
- 查看结构化情报包
- 跳转监控源管理
- 查看监控批次、批次详情与关联公告 run

### 投递中心
- 管理邮件、企业微信、Webhook 目标
- 测试发送目标、查看投递记录
- 执行立即发送、计划发送与失败重试

### 系统工具页
- 查看平台任务与 attempt 轨迹
- 查看 API / Worker / Scheduler / Database 状态

## ✅ 当前仓库状态

当前仓库已经从“文档先行”进入“持续实现 + 持续回写文档”阶段：

- 已完成 Phase 0：数据库分层 / FK、运行拓扑、实现顺序口径收口
- 已完成 Phase 1：`backend/ + frontend/ + infra/` 工程骨架落地
- 已完成 Phase 2 的最小平台执行底座与 CVE fast-first 主链
- 已完成 CVE Phase A：官方优先多源 seed、非 NVD 规则库、Bugzilla 单页提取与 regression fixtures
- 已完成 CVE Phase B2：seed 直达 patch/commit 优先消费、页面抓取局部容错、工作台终态后历史同步刷新
- 已完成平台首页摘要、系统状态页、任务中心的真实数据接入
- 已完成安全公告手动提取、监控批次列表与批次详情查询切片
- 已完成投递中心目标管理、配置脱敏展示、测试发送、立即发送、计划发送与失败重试
- 已具备统一验证入口：`scripts/verify_*`、`make phase1-verify`、后端 pytest 与前端 Vitest / build

补充说明：

- 当前 CVE 真实主线仍是实时 fast-first 运行链，不是已落地 graph runtime。
- 当前已具备详情页 `fix family` 聚合视图，但这不是 graph runtime 或 family 持久化模型。
- 安全公告监控当前已落地“监控批次可查、批次详情可看、关联公告 run 可追”，但不应表述为完整自动调度平台已经成熟。
- `LLM fallback` 与离线 `cvelistV5` 基线都不属于当前已实现事实。
