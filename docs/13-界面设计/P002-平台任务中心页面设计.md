# P002 平台任务中心页面设计

> **对应模块：M002 平台任务执行与调度中心**

---

## 🎯 页面目标

`/system/tasks` 是平台内部的任务排障页，负责让用户查看任务状态、attempt 轨迹，并对失败任务执行平台级重试。

它不是场景主入口，但必须提供稳定的运行排障能力。

---

## 🚪 入口与出口

### 入口

- 首页最近任务点击进入
- 工具导航进入 `系统 -> 任务中心`
- 直接访问 `/system/tasks`

### 出口

- 点击某条任务关联的场景结果，跳转对应详情页
- 返回首页或系统状态页

---

## 🧱 页面布局

```text
Topbar
  -> Task Hero
  -> Filter Bar
  -> Task List
  -> Task Detail Drawer
  -> Attempt Timeline
```

### 区块1：Task Hero

- 页面标题
- 平台语义说明：这里只查看执行状态，不替代场景结果页

### 区块2：筛选栏

- 场景筛选
- 状态筛选
- 触发方式筛选

### 区块3：任务列表

- `job_id`
- 场景
- 任务类型
- 状态
- 最近 attempt 时间
- 重试按钮

### 区块4：任务详情抽屉

- 任务基础信息
- 关联 run_id
- 输入快照摘要
- attempt 列表

### 区块5：Attempt 时间线

- 每次 attempt 的开始/结束
- 错误原因
- 当前 attempt 是否成功

---

## 🖱️ 关键交互

- 点击任务行展开详情抽屉，不离开任务页。
- 点击重试只创建新的 attempt，不新建 run。
- 点击关联的 `run_id` 时跳到场景详情页，而不是页面内复制场景结果。

---

## 🎭 状态稿

### 默认态

- 显示最近任务列表，失败态与运行中优先靠前。

### 空态

- 当前还没有平台任务记录。

### 详情加载态

- 抽屉骨架屏，列表保持可见。

### 重试中

- 当前行显示 `queued` 或 `retrying` 过渡态。

### 重试失败

- 在任务详情抽屉显示 attempt 创建失败原因。

---

## 📦 页面视图对象

### `TaskJobListItemView`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `job_id` | string | 任务 ID |
| `scene_name` | string | 场景名 |
| `job_type` | string | 任务类型 |
| `status` | string | 当前状态 |
| `trigger_kind` | string | 触发方式 |
| `scene_run_id` | string | 对应场景 run |
| `last_attempt_at` | string | 最近 attempt 时间 |

### `TaskJobDetailView`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `job_id` | string | 任务 ID |
| `scene_run_id` | string | 场景 run |
| `payload_summary` | object | 输入快照摘要 |
| `attempts` | array | attempt 列表 |

---

## 🔌 API 与字段映射

| 页面区块 | API | 主要字段 |
|----------|-----|----------|
| 任务列表 | `GET /api/v1/platform/tasks` | 列表项字段 |
| 任务详情 | `GET /api/v1/platform/tasks/{job_id}` | `scene_run_id`、attempts |
| 任务重试 | `POST /api/v1/platform/tasks/{job_id}/retry` | 重试结果 |

---

## 🪞 参考约束

- 任务中心属于工具页，不进入一级主导航。
- 页面可以信息密度较高，但不能复制场景详情页的证据展示职责。
