# P003 平台投递中心页面设计

> **对应模块：M003 平台投递目标与投递记录**

---

## 🎯 页面目标

`/deliveries` 是平台统一投递中心，负责管理投递目标、测试发送、查看投递记录和重试失败投递。

---

## 🚪 入口与出口

### 入口

- 工具导航进入 `投递中心`
- 首页最近投递点击进入记录视图
- 公告结果页投递区块点击“进入投递中心”

### 出口

- 返回首页
- 返回安全公告结果页

---

## 🧱 页面布局

```text
Topbar
  -> Delivery Hero
  -> Tab Switch
  -> Targets Panel
  -> Records Panel
  -> Target Editor Drawer
```

### 区块1：Delivery Hero

- 页面标题
- 简短说明：目标管理与投递记录共用一个平台页

### 区块2：Tab 切换

- `目标管理`
- `投递记录`

### 区块3：目标管理面板

- 目标列表
- 渠道类型
- 启用状态
- 新建/编辑
- 测试发送

### 区块4：投递记录面板

- 记录列表
- 状态筛选
- 场景筛选
- 渠道筛选
- 重试按钮

### 区块5：目标编辑抽屉

- 基础字段
- 渠道特有字段
- 测试发送动作

---

## 🖱️ 关键交互

- `tab=records` 通过 URL 持久化。
- 测试发送属于目标级动作，与正式投递分开记录。
- 失败记录的重试只重发本次投递，不触发场景重新执行。

---

## 🎭 状态稿

### 默认态

- 默认打开 `目标管理`。

### 空态

- 没有目标：引导先创建第一个目标。
- 没有记录：显示“当前还没有投递记录”。

### 测试发送中

- 当前目标行显示发送中，不阻断其他操作。

### 记录重试中

- 当前记录行展示重试状态，保留原错误历史。

### 配置错误态

- 字段级错误 + 表单级错误同时可见。

---

## 📦 页面视图对象

### `DeliveryTargetView`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `target_id` | string | 目标 ID |
| `name` | string | 目标名称 |
| `channel_type` | string | 渠道类型 |
| `enabled` | boolean | 是否启用 |
| `config_summary` | object | 配置摘要 |

### `DeliveryRecordView`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `record_id` | string | 记录 ID |
| `scene_name` | string | 来源场景 |
| `status` | string | 投递状态 |
| `source_ref_type` | string | 来源对象类型 |
| `source_ref_id` | string | 来源对象 ID |
| `error_message` | string | 错误摘要 |

---

## 🔌 API 与字段映射

| 页面区块 | API | 主要字段 |
|----------|-----|----------|
| 目标列表 | `GET /api/v1/platform/delivery-targets` | 目标字段 |
| 新建目标 | `POST /api/v1/platform/delivery-targets` | 目标对象 |
| 更新目标 | `PATCH /api/v1/platform/delivery-targets/{target_id}` | 目标对象 |
| 测试发送 | `POST /api/v1/platform/delivery-targets/{target_id}/test` | 测试结果摘要 |
| 记录列表 | `GET /api/v1/platform/delivery-records` | 记录字段 |
| 记录重试 | `POST /api/v1/platform/delivery-records/{record_id}/retry` | 重试结果 |

---

## 🪞 参考约束

- 投递中心属于平台工具页，不用后台模板墙式布局。
- `目标管理` 与 `投递记录` 必须放在一个连续体验里，避免两个孤立页面。
