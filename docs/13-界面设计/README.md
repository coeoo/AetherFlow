> **AetherFlow 页面级 UI 规格入口**

> 读前建议：先阅读 `../00-总设计/总体项目设计.md`、`../03-系统架构/前端架构设计.md` 和 `../04-功能设计/README.md`。本目录负责页面级信息架构、线框、交互状态与视觉继承策略，不承担系统总纲、数据库契约或场景领域定义。

---

## 🎯 目录职责

`13-界面设计/` 是当前仓库新增的页面级设计层，用来补齐“模块设计已经存在，但页面还不够可交付”的空档。

本目录只回答四类问题：

1. 页面之间如何组织与跳转。
2. 每个页面应该长什么样、有哪些区块。
3. 页面在加载、空态、失败、终态时如何表现。
4. 备份仓的 UI 资产哪些继承、哪些清理、哪些不再进入新仓。

不在本目录解决的问题：

- 场景结果模型的最终领域定义
- 数据表与索引设计
- 后端任务、调度和重试契约
- 高保真视觉系统或组件库 API

## 🔗 与模块设计的关系

页面规格与 `04-功能设计/` 的关系固定如下：

- `M001` 对应 `P001`
- `M002` 对应 `P002`
- `M003` 对应 `P003`
- `M005` 对应 `P005`
- `M101` 对应 `P101`
- `M102` 对应 `P102`
- `M201` 对应 `P201`
- `M202` 对应 `P202`
- `M203` 对应 `P203`
- `M204` 对应 `P204`
- `M205` 对应 `P205`

约束：

1. `Mxxx` 文档负责业务目标、数据对象、接口和测试要点。
2. `Pxxx` 文档负责页面布局、状态稿、区块层级和交互路径。
3. 如果页面需要而模块文档未承诺字段，应先回填 `Mxxx`，再进入实现。

## 🧱 页面规格模板

每个页面文档必须包含以下章节：

1. 页面目标与目标用户
2. 入口、出口与跳转关系
3. 页面布局与区块层级
4. 关键交互动作
5. 状态稿
6. 页面视图对象
7. API 与字段映射
8. 参考资产与继承约束
