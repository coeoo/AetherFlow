# P005 平台系统状态页面设计

> **对应模块：M005 平台系统配置与健康观测**

---

## 🎯 页面目标

`/system/health` 负责展示系统健康摘要、关键依赖状态和运行级告警提示，为首页健康卡片提供完整落点。

---

## 🚪 入口与出口

### 入口

- 首页健康摘要点击进入
- 工具导航进入 `系统 -> 系统状态`
- 直接访问 `/system/health`

### 出口

- 返回首页
- 进入任务中心

---

## 🧱 页面布局

```text
Topbar
  -> Health Hero
  -> Core Status Grid
  -> Dependency Notes
  -> Recent Runtime Alerts
```

### 区块1：Health Hero

- 页面标题
- 当前系统等级：healthy / degraded / down

### 区块2：核心状态卡

- API
- Database
- Worker
- Scheduler

### 区块3：依赖说明

- 最近异常摘要
- 当前启用的来源/渠道摘要

### 区块4：运行告警

- 最近失败说明
- 建议跳转到任务中心或相关场景

---

## 🖱️ 关键交互

- 点击某个异常项可跳到任务中心或留在本页查看说明。
- 系统状态页不展示复杂图表，只展示当前可行动的信息。

---

## 🎭 状态稿

### 正常态

- 四类核心状态全部健康，说明区简洁。

### 降级态

- 某个关键组件异常时，顶部明确显示 `degraded`。

### 故障态

- 数据库或 API 不可达时显示 `down`，并弱化无意义的次级区块。

### 摘要失败态

- 健康摘要接口失败时显示全页错误态，但保留返回首页入口。

---

## 📦 页面视图对象

### `HealthSummaryView`

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `api` | string | API 状态 |
| `database` | string | 数据库状态 |
| `worker` | string | Worker 状态 |
| `scheduler` | string | Scheduler 状态 |
| `notes` | array | 说明与告警 |
| `enabled_sources` | array | 启用来源摘要 |
| `enabled_channels` | array | 启用渠道摘要 |

---

## 🔌 API 与字段映射

| 页面区块 | API | 主要字段 |
|----------|-----|----------|
| 存活探测 | `GET /api/v1/platform/health` | 基础状态 |
| 页面摘要 | `GET /api/v1/platform/health/summary` | 完整摘要对象 |

---

## 🪞 参考约束

- 系统页是工具页，不做监控大屏。
- 页面要比首页健康卡片更完整，但不演变成运维平台。
