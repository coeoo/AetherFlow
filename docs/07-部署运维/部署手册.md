# 部署手册

> **AetherFlow 部署与运行说明**

---

## 1. 部署组件

v1 包含以下进程：

1. `web`：前端静态资源服务
2. `api`：FastAPI 接口服务
3. `worker`：后台任务执行器
4. `scheduler`：定时任务调度器
5. `postgres`：主数据库

---

## 2. 开发环境启动顺序

1. 启动 PostgreSQL
2. 执行数据库初始化 SQL
3. 启动 API
4. 启动 Worker
5. 启动 Scheduler
6. 启动前端开发服务器

---

## 2.1 前端开发约束

- 开发环境优先使用 Vite dev server
- 推荐通过 `VITE_DEV_PROXY_TARGET` 把 `/api` 代理到后端 API
- 页面调试时至少验证：
  - `/`
  - `/cve`
  - `/announcements`
  - `/deliveries`
  - `/system/health`

---

## 3. 生产环境部署建议

- API、Worker、Scheduler 分离部署
- 前端静态资源单独发布
- Artifact 目录使用独立挂载卷
- PostgreSQL 独立实例部署

---

## 3.1 前端构建与发布

建议流程：

1. 安装前端依赖
2. 执行前端构建，产出 `dist/`
3. 由 `web` 进程或 Nginx 等静态服务托管 `dist/`
4. 把 `/api` 反向代理到 FastAPI
5. 为 SPA 路由启用 `index.html` 回退

发布约束：

- 生产环境推荐前端与 API 同域
- 不直接暴露 Artifact 磁盘目录
- 静态资源可走长期缓存，但 `index.html` 应保守缓存

---

## 4. 数据库初始化

按以下顺序执行：

1. `05-数据库/sql/2026-04-09_init_platform_core.sql`
2. `05-数据库/sql/2026-04-09_init_cve.sql`
3. `05-数据库/sql/2026-04-09_init_announcement.sql`
4. `05-数据库/sql/2026-04-09_init_indexes.sql`

---

## 5. 运行检查

部署完成后至少验证：

- 首页可访问
- `/announcements?tab=monitoring` 可访问
- `/deliveries` 可访问
- `/system/health` 可访问
- `/api/v1/platform/health` 返回正常
- Worker 能消费手动任务
- Scheduler 能扫描监控源
- 投递中心可创建测试目标

---

## 6. 回滚原则

- 数据库结构变更必须有对应迁移或回滚脚本
- 投递渠道配置变更要保留旧版本记录
- 新来源适配器上线前应先在试跑模式验证
