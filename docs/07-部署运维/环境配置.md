# 环境配置

> **AetherFlow 环境变量与运行配置说明**

---

## 0. 前端 / Web 配置

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `VITE_APP_TITLE` | 否 | 浏览器标题，默认 `AetherFlow` |
| `VITE_API_BASE_URL` | 开发环境建议必填 | 前端访问 API 的基地址；生产环境推荐同域 `/api` |
| `VITE_BASE_PATH` | 否 | 前端部署子路径 |
| `VITE_DEV_PROXY_TARGET` | 开发环境可选 | Vite 开发代理目标 |
| `WEB_DIST_DIR` | 否 | 前端构建输出目录，默认 `dist` |
| `WEB_PUBLIC_BASE_URL` | 否 | 对外访问基地址 |

**说明**：
- 生产环境优先使用同域部署，减少浏览器跨域与 Cookie/认证歧义。
- Artifact 仍通过 API 间接访问，不新增前端直连磁盘目录配置。

---

## 1. 基础配置

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `AETHERFLOW_ENV` | 是 | 运行环境：`dev/test/prod` |
| `AETHERFLOW_DEBUG` | 否 | 是否开启调试模式 |
| `AETHERFLOW_HOST` | 是 | API 监听地址 |
| `AETHERFLOW_PORT` | 是 | API 端口 |

---

## 2. 数据库配置

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `DATABASE_URL` | 是 | PostgreSQL 连接串 |
| `DATABASE_POOL_SIZE` | 否 | 连接池大小 |
| `DATABASE_MAX_OVERFLOW` | 否 | 溢出连接数 |

---

## 3. Artifact 存储

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `ARTIFACT_ROOT_DIR` | 是 | Artifact 根目录 |
| `ARTIFACT_BASE_URL` | 否 | 对外访问基地址 |

---

## 4. Worker / Scheduler

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `WORKER_CONCURRENCY` | 否 | Worker 并发数 |
| `SCHEDULER_POLL_INTERVAL` | 否 | 调度扫描周期 |
| `TASK_DEFAULT_TIMEOUT_SECONDS` | 否 | 单任务默认超时 |

---

## 5. CVE 场景配置

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `CVE_RUN_MODE_DEFAULT` | 否 | 默认运行模式 |
| `CVE_MAX_FRONTIER_HOPS` | 否 | 最大探索深度 |
| `CVE_FETCH_TIMEOUT_SECONDS` | 否 | 页面抓取超时 |
| `CVE_PATCH_DIR` | 是 | patch Artifact 保存目录 |
| `AETHERFLOW_CVE_LLM_FALLBACK_ENABLED` | 否 | 是否开启 CVE 受限 LLM fallback，默认关闭 |
| `AETHERFLOW_CVE_LLM_FALLBACK_MAX_CANDIDATES` | 否 | 允许送入 fallback 的最大候选数，超过则跳过 |

---

## 6. 安全公告场景配置

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `ANNOUNCEMENT_FETCH_TIMEOUT_SECONDS` | 否 | 抓取超时 |
| `ANNOUNCEMENT_MAX_ITEMS_PER_RUN` | 否 | 每次抓取上限 |
| `WECHAT_STORAGE_STATE_PATH` | 视情况 | 微信源登录态文件 |
| `NCC_PLAYWRIGHT_TIMEOUT_SECONDS` | 否 | NCC 动态抓取超时 |

---

## 7. 模型配置

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `LLM_BASE_URL` | 否 | OpenAI 兼容接口地址 |
| `LLM_API_KEY` | 否 | API Key |
| `LLM_DEFAULT_MODEL` | 否 | 默认模型 |
| `LLM_TIMEOUT_SECONDS` | 否 | 调用超时 |

补充约束：

- 当前 LLM 配置只作为通用 OpenAI 兼容调用配置；是否真正用于 CVE fallback，还取决于 `AETHERFLOW_CVE_LLM_FALLBACK_ENABLED`
- CVE fallback 第一版只允许做候选选择型建议，不自动补下载，不作为主结论来源

## 8. 投递配置

### 邮件
- `SMTP_HOST`
- `SMTP_PORT`
- `SMTP_USERNAME`
- `SMTP_PASSWORD`
- `SMTP_FROM`

### 微信/企业微信
- `WECOM_WEBHOOK_BASE_URL`
- `WECOM_DEFAULT_MENTIONED_LIST`

### Webhook
- `WEBHOOK_DEFAULT_TIMEOUT_SECONDS`
- `WEBHOOK_DEFAULT_RETRY_COUNT`

---

## 9. 部署补充约束

- 前端静态资源默认构建到 `WEB_DIST_DIR`
- Web 服务必须支持 SPA 路由回退到 `index.html`
- 推荐反向代理 `/api` 到后端 API，而不是让浏览器直接跨域访问
